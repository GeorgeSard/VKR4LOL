# Синтетический датасет авиарейсов РФ для ВКР

Файл: `flight_delays_ru_synthetic_2023_2025.csv`

Датасет сформирован для учебной ВКР на тему: «Разработка модели искусственного интеллекта для прогнозирования задержек авиарейсов и классификации их причин».

Важно: датасет является **синтетическим**. Он не содержит реальные коммерческие данные авиакомпаний и не должен использоваться как источник фактической статистики по авиаперевозкам. Цель набора данных — демонстрация полного ML-процесса: загрузка данных, очистка, feature engineering, версионирование DVC, обучение моделей, логирование экспериментов в MLflow и демонстрация через FastAPI.

## Общая характеристика

- Период: 2023-01-01 — 2025-12-31
- Количество строк: 120,360
- Количество уникальных `flight_id`: 120,000
- Намеренно добавленные дубли для демонстрации очистки: 360
- Количество колонок: 36
- Формат: CSV, UTF-8 with BOM для удобного открытия в Excel

## Распределение по годам

- 2023: 40,097 строк
- 2024: 40,192 строк
- 2025: 40,071 строк


## Целевые переменные

### 1. `is_significant_delay`

Первая ML-задача — прогноз операционно значимой задержки авиарейса.

Правило формирования целевой переменной:

```text
delay_minutes >= 15 → is_significant_delay = 1
delay_minutes < 15  → is_significant_delay = 0
```

В этом проекте задержка меньше 15 минут считается небольшим отклонением от расписания, а не положительным классом ML-задачи. Это сделано, чтобы модель прогнозировала именно операционно значимые задержки, а не случайный шум в несколько минут.

Количество значимых задержек: 21,928
Доля значимых задержек: 18.22%

### 2. `delay_reason`

Вторая ML-задача — классификация причины задержки.

Для рейсов без значимой задержки значение `delay_reason` равно `none`. Модель классификации причин следует обучать только на строках:

```text
is_significant_delay == 1
delay_reason != "none"
```

## Колонки, которые нельзя подавать как признаки для первой модели

```text
delay_minutes
is_significant_delay
delay_reason
```

Причина: это целевые/результирующие переменные. Их использование как входных признаков приведёт к утечке данных и искусственно завышенным метрикам.
